# Example Outputs

Reference-caliber examples of the core document types this plugin
produces. Agents reading these in advance of a drafting task tend to
match the format, tone, and structural conventions better than they
would from instructions alone.

## What's here

| File | Document type | Use it when drafting |
|------|---------------|----------------------|
| `example-answer.md` | Answer + affirmative defenses + counterclaim | `wa-first-30-days`, `wa-draft-motion` |
| `example-motion-to-compel.md` | Motion to compel + memorandum | `wa-draft-motion` + `wa-discovery` |
| `example-declaration.md` | Declaration in support with exhibits | `wa-draft-declaration` |
| `example-proposed-order.md` | Proposed order on motion to compel | `wa-draft-order` |
| `example-meet-and-confer.md` | CR 26(i) / CRLJ 26(f) meet-and-confer letter | `wa-discovery` |
| `example-certificate-of-service.md` | Certificate of service | `wa-file-packet` |

## Provenance

These examples are **synthetic** — illustrative, drawn from the
patterns this plugin covers, and not filed in any actual case.
Names, cause numbers, and dates are fictional.

## How agents should use them

1. Before drafting a real document, **read the matching example**
2. Match the **caption format**, **section order**, and **prose
   register**
3. Adapt the **substantive content** to the user's actual facts —
   do not carry over synthetic facts
4. Preserve the **GR 14 formatting cues** (3-inch top margin on
   page 1, signature block, caption layout)
