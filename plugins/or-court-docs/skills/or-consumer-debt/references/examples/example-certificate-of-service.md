## Key features

### UTCR 1.090 requirements

A Certificate of Service must include:

1. **Date of service** — when the documents were sent
2. **Names of documents served** — list the documents
3. **Identity of recipients** — each party served
4. **Method of service** — eService, mail, hand, email, etc.
5. **Address (or email) of each recipient** — for the
   service record
6. **Signature of the server** — with role and contact info

### Methods of service (ORCP 9 B)

| Method | When usable |
|--------|-------------|
| eService via File and Serve | For registered parties |
| Email | If party consented or eService is set up |
| USPS first-class mail | Universally acceptable |
| USPS Certified Mail with Return Receipt | Belt-and-suspenders backup |
| Hand delivery | Universally acceptable |
| Fax | If party consented |

For pro se filers, "eService + Certified Mail" is the
cleanest combo — eService is fast and electronic; Certified
Mail provides written proof of mailing.

### Method-specific entries

The "Method" column should reflect what was actually done:

- "eService via Oregon File and Serve" — for File and Serve
  eService
- "Email to [address]" — for direct email
- "USPS First-Class Mail to [address]" — for ordinary mail
- "USPS Certified Mail with Return Receipt No. [number] to
  [address]" — for certified mail
- "Hand delivery on [date]" — for hand service

### Multiple methods

It's acceptable (and sometimes wise) to serve by multiple
methods:

> Plaintiff's counsel: eService via Oregon File and Serve
> (also USPS Certified Mail No. 9514...)

This protects against eService failure or claims of non-
receipt.

### Signature

The Certificate must be signed by the server — the person
who actually performed the service. For a pro se filer
self-serving, that's the filer themselves.

For an attorney, the signature line typically reflects
counsel of record.

### Statewide use

UTCR 1.090 applies in every Oregon circuit court. The
Certificate format is statewide; local SLR generally does
not modify it.

## Common Certificate-of-Service errors

| Error | Consequence |
|-------|-------------|
| Wrong date | Suggests service didn't happen when claimed |
| Missing party | Service deemed incomplete; clerk may reject |
| Wrong method | Unenforceable; opposing can claim non-receipt |
| No signature | UTCR 1.090 violation; clerk may reject |
| Stale address | Service to wrong address may not be valid |
| Forgetting the eService receipt | eService is sufficient as proof; document it |

## Pro se cleanliness tip

For pro se filers, maintaining a parallel **Service Log**
spreadsheet is invaluable:

```
| Date       | Filing                  | Method                                | Recipient   | Tracking/Reference        | Confirmed |
|------------|-------------------------|---------------------------------------|-------------|---------------------------|-----------|
| 04/01/2025 | First RFPs              | eService + Certified                  | R. Smith    | Cert: 9514...4567         | 04/04/2025 |
| 05/10/2025 | Meet-and-confer letter | Email                                  | R. Smith    | Email msg 1283            | 05/10/2025 |
| 06/14/2025 | Motion to Compel       | eService + Certified                  | R. Smith    | Cert: 9514...5678         | 06/16/2025 |
```

If service is later contested, the Service Log + the
Certificates of Service (one for each filing) provide a clean
record.
