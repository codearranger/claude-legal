# or-consumer-debt — Chain-of-title discovery

## Prompt

Help me write RFPs to force the debt buyer to produce its
chain-of-title documents. The case is in Multnomah, the
debt buyer is LVNV Funding, and the original creditor was
Bank of America.

## Expected triggers

- `or-consumer-debt`
- `or-discovery`
- `or-multcc`

## Acceptance criteria

### RFPs targeting chain of title

- [ ] Original Cardholder Agreement
- [ ] Bills of Sale at each transfer in the chain
- [ ] Assignment Schedules at each transfer (unredacted, with
      narrow-row redaction option)
- [ ] Bulk Sale Agreements
- [ ] Account history (post-charge-off)

### RFPs targeting authentication

- [ ] Custodian declarations or OEC 902(11) certifications
- [ ] Witness lists for trial authentication

### RFPs targeting ORS 697

- [ ] LVNV's DCBS registration documents under ORS 697.015

### RFPs targeting communications and validation

- [ ] All communications between Plaintiff and Defendant
- [ ] Validation notice records
- [ ] Dispute response records

### Drafting principles

- [ ] Uses "ACCOUNT", "PLAINTIFF", "ORIGINAL CREDITOR" as
      defined terms
- [ ] Specific and bounded requests
- [ ] References ORCP 36 B(1), ORCP 43

### Multnomah-specific

- [ ] References Multnomah SLR 5.045 meet-and-confer
      certification requirement for any future motion to
      compel

### What it should NOT include

- [ ] Should NOT include "interrogatories" (not available in
      Oregon ORCP)
- [ ] Should NOT cite WA CR 33 or CR 34
- [ ] Should NOT cite KCDC procedures

## Common failure modes

- Generic "produce all documents related to the account"
- Including interrogatories (Oregon doesn't allow them)
- Failing to address authentication / OEC 902(11)
- Forgetting the DCBS registration target
