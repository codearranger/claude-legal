# az-draft-order — Proposed order and lodging under Ariz. R. Civ. P. 58

## Prompt

The judge granted my motion in Arizona Superior Court and told me
to "submit a proposed order." I'm self-represented. How do I draft
the order and get it to the judge the right way?

## Expected triggers

- `az-draft-order`
- `az-statewide-format`
- `az-submit-order`

## Acceptance criteria

### Document identification

- [ ] Identifies the **proposed order** and explains the **lodging**
      procedure under **Ariz. R. Civ. P. 58** (form of judgments /
      orders; submission for the court's signature)
- [ ] Explains the lodge-and-notice mechanics — the opposing party's
      opportunity to object to the form of the order before the
      court signs — reading the current day counts and procedure
      from the references corpus rather than asserting them from
      memory
- [ ] Distinguishes a proposed order (drafted by a party, signed by
      the judge) from the party's motion; the proposed order
      contains decretal language, not argument

### Caption and structure

- [ ] Caption per **Ariz. R. Civ. P. 10**; title identifies it as a
      proposed Order (or Judgment) with a signature line and date
      line for the judge
- [ ] Decretal "IT IS ORDERED" language tracking the relief granted;
      no factual recitations beyond what the ruling supports
- [ ] If a final judgment, notes the **Rule 54(b) / 54(c)** finality
      language consideration (read the current requirement from the
      corpus) rather than asserting it from memory

### Composition

- [ ] Self-represented preparer block; no bar number
- [ ] Cross-references `az-submit-order` for transmittal mechanics

## Common failure modes

- Confuses lodging a proposed order with filing a motion
- Asserts the objection/lodging day counts from memory
- Puts argument into the order instead of decretal language
- Omits finality-language consideration where the order is a final
  judgment
