# Proposed Order Template (California)

A proposed order for judge signature in a California superior
court. Compliant with CRC 2.100-2.119 and CRC 2.111. Per CRC
2.112, the title must begin with "[PROPOSED]" to distinguish
it from a signed order.

## Structure

1. Attorney/party info block (upper-left, within 2.5")
2. Court name (centered)
3. Caption (party block + case number + "[PROPOSED] ORDER..." title)
4. Horizontal rule
5. Recital clause (what the court considered)
6. Findings (optional, if findings are required)
7. Order clause ("IT IS ORDERED that...")
8. Signature block for the judge
9. Space for date

## Rules for proposed orders

### Title: "[PROPOSED]" in brackets

Per CRC 2.112, include "[PROPOSED]" at the beginning of the
order title so the clerk and parties can distinguish the
proposed order from the signed order. Once the judge signs,
the clerk typically stamps or marks "SIGNED" on the document.

### Deliver or eFile as part of the motion papers

The proposed order is filed simultaneously with the motion
as a separate PDF. Most California eFiling portals have a
specific document code for "Proposed Order."

### Include all relief requested

Draft the proposed order to grant all of the relief requested
in the motion. It is standard practice to also include an
alternative "DENIED" clause (or to draft a separate proposed
order for each possible outcome) — but most practitioners
include only the "GRANTED" version and rely on the court to
modify if partially granting.

### Complex relief — use numbered paragraphs

For orders with multiple elements (compel responses AND award
sanctions AND set a compliance deadline), use numbered
paragraphs for clarity. Each paragraph should be a single,
complete directive.

### Findings clause (Code Civ. Proc., § 437c)

For summary judgment orders, California law requires specific
findings under Code Civ. Proc., § 437c(g):

```
      The Court finds:

      1.   There is no triable issue of material fact as to
           [claim/defense].

      2.   Defendant is entitled to judgment as a matter of
           law because [brief statement of legal basis].
```

For routine discovery orders, findings are not required, but
a brief recital ("good cause appearing") is standard.

### For orders to show cause (OSC)

If the proposed order is an Order to Show Cause (OSC), use
this structure:

```
      IT IS ORDERED that Plaintiff Velocity Investments, LLC,
      shall appear and show cause, if any there be, why
      [relief] should not be granted, on:

      Date:  ___________________
      Time:  ___________________
      Dept.: ___________________

      Any written response by Plaintiff must be filed and
      served no later than [date].
```

### After the hearing — submitting the signed order

After the judge announces or issues the ruling (by tentative
or at hearing), the moving party is responsible for:

1. Revising the proposed order to match the actual ruling
2. eFilings the revised proposed order through the eFiling portal
3. For LASC: submit via the department or through the eFiling
   system as "Proposed Order for Signature"
4. For SFSC: submit through File & ServeXpress to Dept. 302

Some judges' standing orders require submission of the proposed
order within 5 days of the hearing; others within 10 days.
Check the standing order for the specific department.

## Proof of Service of signed order

Once the order is signed and file-stamped, it must be served
on all parties. File a Proof of Service (POS-030) confirming
that you served the signed order. This starts the clock for
any deadlines triggered by the order (e.g., "within 20 days
of the date of this Order").
